# Тестове завдання: backend

Ми створюємо мобільні додатки та іноді для тестування гіпотез доводиться запускати АВ-тести.
Для цього нам потрібна система, яка являє собою найпростішу REST API, яке состоїть з одгного ендпоінту.

## API та розподілення
Модільний додаток при запуску генерує деякий унікальний ID клієнта (який зберігається між сесіями) і запитує список експерементів, дадаючи HTTP-заголовок
'Device-Token'. У віфдповідь сервер віддає список еусперементів. Для кожного експеременту клієнт отримує:

* ключ: ім'я експеременту. В клієнті є код, який буде змінювати якусь поведінку в залежності від значення цього ключа.
* значення: строка, одна з можливих опцій (див. нижче).

Важливо, щоб девайс потрапляв в одну групу і завжди залишався в ній.

## Експеременти
### 1. Колір кнопки

У нас є гіпотеза, що колір кнопки "купити" впливає на колнверсію у покупку.

*Ключ: `button_color`
*Опції:
 * `#FF0000` -> 33.3%
 * `#00FF00` -> 33.3%
 * `#0000FF` -> 33.3%

Так після 600 запитів к АРІ з різними 'Device-Token' кожен колір повинні отримати по 200 девайсів.

### 2. Ціна покупки

У нас є гіпотеза, що зміна вартості покупки у додатку може впливати на наш маржинальний прибуток.
Але для того щоб не втрачати кошти, у випадку невдалого експеременту, 75% юзерів будть оримувати стару ціну і тільки на малій частині аудиторії
ми протустуємо оновлення:
* Ключ `price`
* Опції:
 * `10` -> 75%
 * `20` -> 10%
 * `50` -> 5%
 * `5` -> 10%

# Вимоги та обмеження
1. Якщо девайс одного разу отримав значення, то він завжди буде отримувати тільки його.
2. Експеримент проводиться тільки для нових девайсів: якщо експеримент створено після першого запиту від девайсу, то девайс не повинен нічого знати про цей експеримент.

## Завдання

1. Спроектуйте, опишіть та реалізуйте АРІ.
2. Додайте експерименти (1) та (2) у додаток.
3. Створіть сторінку для статистики: проста таблиця зі списком експерементів, загальна кількість девайсів які приймають участь у експеременті та їх розповсюдження між опціями

Можн авикористовувати будь-які технології та бібліотеки

Буде полюсом:

* Наявність тестування
* Задеплоїна версія додатку
* Швідкість відповіді від серверу <100ms
